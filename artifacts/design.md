# WFTI H5 Design

## 1. 设计总述
- 项目：世界杯球迷人格大鉴赏（WFTI）H5 活动
- 设计载体：H5 / mobile-web / App WebView
- 已锁定视觉风格：A 高阶 3D 风格建模
- 本轮目标：在保留高亮影棚、390px KV、底色同步、底部防真空与商业级高级感的前提下，完成一次以“真实中文文案承载能力”为优先的版式整改

本方案基于以下输入修正：
- `artifacts/source-prd/prd.md`
- `artifacts/source-design/design-spec.md`
- `handoffs/2026-04-24-leader-to-designer-design-rework-overflow.md`
- `runs/2026-04-24-leader-request-design-rework-overflow.md`

## 2. 全局修正策略
### 2.1 不变的方向
- 保留 `A 高阶 3D 风格建模`
- 保留高亮影棚、冷暖双光、玻璃卡与镜面地板
- 保留 `390px` 主视觉锚点与底色同步策略
- 保留底部悬浮 CTA 与长屏防真空思路

### 2.2 本轮结构性修正
- 所有页面改为“内容优先定版”，不再先堆装饰后塞中文
- 对带底栏页面统一补充底部安全区，内容区最少预留 `96px` 到 `128px`
- 明确首屏重点、滚动续读区与底部固定操作栏三者关系
- 对长文案增加最大宽度、段落节奏、`line-clamp` 与弱化信息下放策略
- 减少英文占位与概念化标签，统一成成熟中文活动页表达

### 2.3 底色与环境层
- KV 宽度继续锁定为 `390px`
- KV 底部真实采样色维持 `--floor-rgb: 224, 231, 240`
- 全页底色变量统一为 `--page-floor: rgb(var(--floor-rgb))`
- 页面底部额外叠加 18% 珍珠灰至银蓝灰渐变，消除长屏底部真空

### 2.4 字体与中文排版
- 主标题：`34/40`，厚重几何黑体，颜色 `#31455D`
- 人格名：`30/36`
- 模块标题：`18/24`
- 正文：`15/24`
- 长题干与长选项场景允许降到 `16/26` 与 `15/22` 的组合，但不低于可舒适阅读阈值
- 减少大面积全大写英文，保留必要人格代号即可

## 3. 本轮设计审查修正
### 3.1 修正范围
- 活动落地页：重做信息卡节奏，避免说明文案与底栏互相挤压
- 答题页：针对最长题干与最长选项重排题卡和选项卡
- 结果页：从单屏堆满改为“首屏重点 + 下滑续读 + 固定底栏”
- 分享海报态：减少文案量，稳住二维码区与视觉重心
- 异常态：缩短界面展示文案并增加最大宽度，防止长句撑坏卡片

### 3.2 已修复的溢出与遮挡问题
- 修复结果页首屏同时容纳插画、标签云、雷达图、解析、三联卡、二维码与底栏导致的裁切
- 修复答题页长题干下选项区被压缩、点击热区不足的问题
- 修复底部悬浮栏覆盖正文与二维码区的问题
- 修复异常态说明文案过长时卡片被拉爆、按钮贴边的问题
- 修复海报态二维码区与长解析文案互相争夺空间的问题

### 3.3 已修复的节奏问题
- 英文眉题从主表达层退到辅助信息层，主阅读链路改回中文
- 落地页从“KV + 一块重卡 + 底栏”改为“KV + 信息摘要卡 + 说明卡 + 固定 CTA”
- 结果页首屏只保留身份结论、角色主视觉、核心标签与下滑提示
- 深度解析、雷达图、关系三联卡、二维码全部下放到滚动内容区

## 4. 页面设计说明
### 4.1 活动落地页
视觉目标：在首屏建立高级感，但不牺牲活动说明与真实文案的承载能力。

首屏展示：
- Logo
- 390px 3D KV
- 主标题 `世界杯球迷人格大鉴赏`
- 单行 Slogan `你不是在看世界杯，你是在演你自己。`
- 核心信息卡：`16 道题 / 3 分钟 / 16 型人格`
- 社交背书
- 固定底栏：`开始鉴赏` / `查看我的结果`

下滑续读：
- 活动说明
- 测试方式与结果产出说明
- 长句说明收口后的界面文案，例如将 PRD 中的 `测出你的世界杯球迷人格密码` 在界面中展示为 `测出你的球迷人格档案`

版式策略：
- 信息卡与 KV 仍保持负边距搭接，但卡内改成两层信息，不再把所有文案塞进同一块玻璃卡
- 底栏以上必须预留安全底距，防止 430x932 下首屏过松或 390x844 下被遮挡

### 4.2 答题页
视觉目标：适配最长题干与最长选项，同时保留舒服的点击热区与答题节奏。

首屏展示：
- 返回按钮
- 品牌胶囊
- 进度条与 `N/16`
- 维度标签
- 题干

下滑续读：
- 3 个长选项卡
- 辅助说明
- `24h 内可继续作答` 提示

版式策略：
- 题目卡从单块厚卡拆成“题头区 + 选项区”，避免标题与选项互抢空间
- 题干最大宽度控制在卡片内部安全宽度内，避免中文长句拉得过散
- 选项卡最小高度提升到可单手点击区间，文本允许两到四行自然换行
- 对特别长的展示文案不缩到难读，而是通过增加滚动高度与卡片纵向节奏解决

最长场景校验：
- 题干按 PRD 第 16 题与第 10 题长度校验
- 选项按 PRD 第 5 题 C 选项、第 13 题 C 选项与第 15 题 C 选项长度校验

### 4.3 结果计算过渡页
视觉目标：保留仪式感，但界面文案更简洁。

首屏展示：
- 3D 人格核心球
- 扫描环
- 状态文案 `正在生成你的球迷人格档案`
- 维度扫描 chips

策略：
- 减少技术感英文标签
- 文案控制在两层内，不增加额外解释段落

### 4.4 结果页
视觉目标：让用户第一眼先认领人格，再通过滚动继续阅读完整档案。

首屏展示：
- 人格编号
- 中文人格名
- 一句话结论
- 角色主视觉
- 4 到 6 个核心标签
- `下滑查看完整人格档案` 提示
- 固定底部操作栏

二屏 / 后续内容：
- 八维雷达图
- 深度人格解析
- 球星对照 / 最佳拍档 / 天敌属性三联卡
- 二维码与分享引导区

结构原则：
- 插画和标签云只服务于首屏结论，不再承担全部信息展示
- 雷达图、长解析、三联卡与二维码全部进入滚动内容区
- 结果页滚动内容底部必须额外预留固定底栏安全空间

界面级文案收口说明：
- PRD 中较长的解析句在评审稿里可收口为更短的 UI 展示文案，例如 `你享受看球时的集体能量，愿意把情绪推到最满，也自然会成为身边球迷的节奏发起者。`
- 该收口仅用于界面评审与版式承载，不改变 PRD 原意

### 4.5 分享海报态
视觉目标：更像成品传播物，而不是结果页截图。

海报信息取舍：
- 保留人格名、短句标签、人格代号、简版雷达表达、二维码与品牌
- 去掉长段解析与多组附加标签

构图策略：
- 上部 46% 给主视觉与人格名
- 中部 26% 给短结论与雷达信息
- 下部 28% 给二维码区与传播引导
- 二维码独立置于留白充足的磨砂盒中，左右至少保留 `24px` 安全距

### 4.6 异常态
视觉目标：统一世界观下的简洁状态页，能承载较长提示但不显拥挤。

展示策略：
- 标题始终单独成层
- 说明文案限制最大宽度，控制在 2 到 3 行
- 长句收口为更像界面文案的表达，例如 `网络恢复后可继续从断点返回`
- 活动已结束页保留热门人格 TOP3，但不再堆多余解释

## 5. 组件与约束
### 5.1 卡片
- 大卡圆角：`32px`
- 中卡圆角：`24px`
- 带底栏页面的主内容区底部内边距：`120px`
- 滚动内容区单模块间距：`14px` 到 `18px`

### 5.2 文案承载规则
- 标题最多 2 行，超出时优先改为更成熟的中文收口文案
- 正文说明最多 3 行，超出使用 `line-clamp`
- 标签云单枚标签不超过 8 个字
- 二维码引导文案不超过两行

### 5.3 底部悬浮栏规范
- 落地页：双按钮或单按钮
- 结果页：三按钮结构，主操作权重最高
- 底栏距底边 `12px` 到 `16px`
- 内容区统一为底栏预留安全空间，不允许正文贴到底栏后面

## 6. 390x844 与 430x932 适配结论
- 390x844：通过减少首屏无效装饰、提升滚动占比与补足底栏安全区，所有页可完整承载真实中文文案
- 430x932：通过保留 390px KV 核心锚点、扩展环境层与内容区纵向留白，页面不会出现上重下轻或底部真空
- 两种比例下均采用同一信息结构，不依赖针对单一机型的特殊裁切
