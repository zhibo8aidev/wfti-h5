# WFTI H5 Design

## 1. 设计总述
- 项目：世界杯球迷人格大鉴赏（WFTI）H5 活动
- 设计载体：H5 / mobile-web / App WebView
- 已锁定视觉风格：A 高阶 3D 风格建模
- 本轮目标：以 `artifacts/source-design/hero-reference.jpg` 为最新 KV 基线，在保留高亮影棚、390px KV、底色同步、底部防真空与真实中文承载能力的前提下，把整套页面统一到同一个“世界杯竞技舞台”视觉世界中

本方案基于以下输入修正：
- `artifacts/source-prd/prd.md`
- `artifacts/source-design/design-spec.md`
- `artifacts/source-design/hero-reference.jpg`
- `handoffs/2026-04-24-leader-to-designer-hero-reference-update.md`
- 既有整改结果：上一轮中文承载与版式修复成果

## 2. Hero 参考图提炼与系统化延展
### 2.1 从头图提炼出的视觉基因
- 主色板：以蓝绿赛场能量为主，辅以高亮白雾地板和少量金色奖杯高光；页面主配色统一为 `蓝 / 绿 / 白 / 金`
- 光感结构：顶部强聚光、中段赛场透视、底部白雾抬亮，形成“舞台灯光打在草地与人物上”的明亮竞技感
- 曲线语言：采用弧形赛场透视线、飘带速度带、内弯光轨来建立冲刺与推进动势
- 地板语言：底部不再只是浅灰背景，而是带有白色雾化和银蓝渐变的“影棚赛场地板”
- 焦点策略：落地页通过人物冲刺、奖杯高光、中心聚焦形成赛事主舞台；非头图页面只保留其抽象语言，不直接复制完整场景

### 2.2 非头图页面如何承接这些基因
- 答题页：用蓝绿渐变、赛场弧线与底部亮雾承接 KV，弱化具象角色，保留“正在场上推进”的节奏感
- 结果计算过渡页：强化中心聚焦、扫描环与金色微高光，让等待过程像在聚光灯下生成球迷人格
- 结果页：用舞台拱形光带、底部雾化地板、局部金色点亮和曲线切线，形成“人物结论页是从 KV 主舞台延展出来的二级舞台”
- 分享海报态：保留蓝绿主场与金色点睛，强调成熟、成品化传播构图，不把头图元素堆满整张海报
- 异常态：只借用高亮赛场底色、柔和弧光和少量金色状态强调，保持简洁、稳定、可读

## 3. 全局修正策略
### 3.1 不变的方向
- 保留 `A 高阶 3D 风格建模`
- 保留高亮影棚与商业级成熟调性
- 保留 `390px` 主视觉锚点与底色同步策略
- 保留长屏底部防真空思路
- 不破坏上一轮已修好的中文文案承载与版式和谐度

### 3.2 本轮结构性修正
- 所有页面从“头图参考图形语言”出发统一视觉系统，而不是只有落地页换一张头图
- 非头图页面改为“抽象延展”，保留蓝绿主调、白雾地板、曲线速度感、局部金色高光，不直接照搬奖杯与人物
- 继续执行“内容优先定版”，所有视觉强化都服从中文阅读链路
- 对带底栏页面统一预留底部安全区，主内容区最少保留 `96px` 到 `128px`

### 3.3 底色与环境层
- KV 宽度继续锁定为 `390px`
- KV 底部真实采样色维持高亮白雾落地后的浅银蓝：`--floor-rgb: 224, 231, 240`
- 全页底色变量统一为 `--page-floor: rgb(var(--floor-rgb))`
- 页面底部叠加白雾到银蓝灰的环境沉淀，避免长屏真空并保持与 KV 地板连续
- 所有页面允许叠加低透明度蓝绿弧光，但数量受控，不制造杂讯

### 3.4 字体与中文排版
- 主标题：`34/40`，厚重几何黑体，颜色从底部环境色中提取，不使用纯黑
- 人格名：`30/36`
- 模块标题：`18/24`
- 正文：`15/24`
- 长题干与长选项场景允许降到 `16/26` 与 `15/22` 的组合，但不低于舒适阅读阈值
- 继续减少大面积英文占位，英文人格代号仅作为辅助信息层

## 4. 本轮设计审查修正
### 4.1 修正范围
- 活动落地页：KV 重构为“蓝绿赛场舞台 + 冲刺人物 + 金色高光 + 白雾地板”的新头图语言
- 答题页：改造成从头图延展出的答题赛道氛围，保留长题干与长选项稳定承载
- 结果页：从单屏堆满改为“首屏舞台认领 + 下滑续读完整档案 + 固定底栏”
- 分享海报态：把头图语言收敛成传播物级构图
- 异常态：从同一视觉世界中抽离出更克制、更安静的状态页版本

### 4.2 已修复的溢出与遮挡问题
- 不因引入新头图语言而破坏结果页首屏、滚动内容区与固定底栏的安全关系
- 不因增加赛场曲线和地板光感而挤压答题页最长题干与最长选项
- 不让底部固定 CTA 覆盖正文、二维码区与状态页按钮
- 不让海报态重新回到信息过满、二维码区被压缩的状态

### 4.3 已修复的节奏问题
- 落地页首屏视觉更强，但说明信息仍拆为摘要卡与说明卡承接
- 结果页首屏只保留人格认领、角色舞台、核心标签与下滑提示
- 深度解析、雷达图、三联卡与二维码继续下放到滚动阅读区
- 异常态继续使用成熟中文表达，不让视觉主导信息

## 5. 页面设计说明
### 5.1 活动落地页
视觉目标：让用户第一眼感受到“世界杯主舞台”而不是普通测试页，同时保持中文说明和 CTA 的清晰承接。

KV 吸收头图的关键语言：
- 蓝绿主色与赛场能量感作为主基调
- 顶部中心聚光与远端赛场弧线构成舞台透视
- 底部白色雾化地板与亮雾承接卡片
- 左侧金色奖杯高光点亮赛事荣誉感
- 右侧冲刺人物形成竞技与推进感
- 速度带、飘带与弧线切线用于拉出冲刺动势

首屏展示：
- Logo
- 390px KV
- 主标题 `世界杯球迷人格大鉴赏`
- 单行 Slogan `你不是在看世界杯，你是在演你自己。`
- 核心信息卡：`16 道题 / 3 分钟 / 16 型人格`
- 社交背书
- 固定底栏：`开始鉴赏` / `查看我的结果`

下滑续读：
- 活动说明
- 测试方式与结果产出说明
- 长句说明收口后的成熟中文界面文案

版式策略：
- KV 下方仍使用负边距搭接，但卡片视觉会从“普通玻璃卡”调整为“赛场雾地上漂浮的亮卡”
- 底栏以上必须预留安全底距，防止 390x844 与 430x932 下首屏关系失衡

### 5.2 答题页
视觉目标：像从 KV 主舞台进入答题赛道，延续同一视觉世界，但降低具象叙事密度，优先保证可答、可读、可点。

如何承接头图基因：
- 继承蓝绿环境色和顶部聚光
- 用抽象弧线、内弯速度带和底部雾化地板替代具象奖杯与人物
- 选项卡高亮时增加少量金色点光，呼应奖杯高光策略

首屏展示：
- 返回按钮
- 品牌胶囊
- 进度条与 `N/16`
- 维度标签
- 题干

下滑续读：
- 3 个长选项卡
- 辅助说明
- `24h 内可继续作答` 提示

版式策略：
- 题目卡保持“题头区 + 选项区”拆分
- 题干最大宽度控制在安全阅读范围内
- 选项卡最小高度维持单手点击舒适区
- 不通过缩小字号去容纳真实中文，而通过纵向节奏与页面高度解决

### 5.3 结果计算过渡页
视觉目标：保持仪式感，但语言上更像“聚光灯下的人格生成”。

如何承接头图基因：
- 中央聚焦光感来自头图中心强光
- 扫描环与粒子沿用赛场弧线逻辑
- 金色高光只作为少量完成感提示，不大面积出现

首屏展示：
- 3D 人格核心球
- 扫描环
- 状态文案 `正在生成你的球迷人格档案`
- 维度扫描 chips

### 5.4 结果页
视觉目标：让用户像站上自己的二级主舞台，先认领人格，再下滑阅读完整档案。

如何承接头图基因：
- 首屏背景使用赛场拱形透视和底部亮雾地板
- 角色主视觉的姿态与曲线语言呼应 KV 冲刺感，但复杂度更收敛
- 4 到 6 个核心标签沿用蓝绿白主调，金色只用来强调人格勋章感

首屏展示：
- 人格编号
- 中文人格名
- 一句话结论
- 角色主视觉
- 4 到 6 个核心标签
- `下滑查看完整人格档案` 提示
- 固定底部操作栏

二屏 / 后续内容：
- 八维雷达图
- 深度人格解析
- 球星对照 / 最佳拍档 / 天敌属性三联卡
- 二维码与分享引导区

结构原则：
- 插画与标签云只服务首屏认领
- 雷达图、长解析、三联卡与二维码全部进入滚动内容区
- 滚动内容底部必须额外预留固定底栏安全空间

### 5.5 分享海报态
视觉目标：像完整传播物，不是结果页截图。

如何承接头图基因：
- 继承蓝绿主场、白雾底板和局部金色徽章感
- 去掉大面积复杂弧线，只保留一到两条大曲线建立构图张力
- 主视觉姿态与人格名形成海报中心，不挤占二维码区

海报信息取舍：
- 保留人格名、短句标签、人格代号、简版雷达表达、二维码与品牌
- 去掉长段解析与多组附加标签

### 5.6 异常态
视觉目标：统一世界观下的简洁状态页，能承载较长提示但不显拥挤。

如何承接头图基因：
- 保留明亮赛场底色、柔和弧光和底部亮雾
- 金色只在“未开始 / 已结束”场景做少量状态强调
- 网络异常页减少高光权重，确保信息最稳定

展示策略：
- 标题始终单独成层
- 说明文案限制最大宽度，控制在 2 到 3 行
- 长句继续收口为成熟界面表达
- 活动已结束页保留热门人格 TOP3，但不堆额外解释

## 6. 组件与约束
### 6.1 卡片
- 大卡圆角：`32px`
- 中卡圆角：`24px`
- 带底栏页面的主内容区底部内边距：`120px`
- 滚动内容区单模块间距：`14px` 到 `18px`
- 卡片材质保持高亮玻璃感，但允许叠加极轻的蓝绿环境色反射

### 6.2 文案承载规则
- 标题最多 2 行，超出时优先改为更成熟的中文收口文案
- 正文说明最多 3 行，超出使用 `line-clamp`
- 标签云单枚标签不超过 8 个字
- 二维码引导文案不超过两行

### 6.3 底部悬浮栏规范
- 落地页：双按钮或单按钮
- 结果页：三按钮结构，主操作权重最高
- 底栏距底边 `12px` 到 `16px`
- 内容区统一为底栏预留安全空间，不允许正文贴到底栏后面

### 6.4 金色高光使用规则
- 金色只用于奖杯感、人格勋章感、主按钮能量边缘与局部高光
- 非头图页面金色面积必须受控，避免变成金蓝混杂的促销风

## 7. 390x844 与 430x932 适配结论
- `390x844`：通过锁定 `390px` KV、保留白雾地板连续性与控制卡片搭接深度，页面可完整承载真实中文文案
- `430x932`：通过扩展环境层、曲线光带和底部沉淀渐变，长屏下依旧不会出现上重下轻与底部真空
- 两种比例下均采用同一信息结构与同一视觉基因，不依赖单机型特供裁切

## 8. 开发落地备注
- mockup 中需明确体现“头图视觉基因提炼”和“非头图页面延展设计”的关系
- 所有页面统一暴露底色变量、蓝绿主色、金色点睛与白雾地板层
- 海报生成时隐藏底部操作栏，仅保留海报专属画布
- 结果页、落地页、异常态继续验证中文长句与系统字号放大后的稳定性
