# PRD — 世界杯球迷人格大鉴赏（WFTI）H5 活动

> **文档版本**：V1.0
> **产品负责人**：待定
> **最后更新**：2026-04-16
> **涉及端**：H5（APP 内 WebView 承载）
> **优先级**：P0

---

## 一、需求背景与目标

- **背景**：2026 世界杯赛期（6~7 月），直播吧需一个高参与度 H5 互动活动承接赛事流量峰值，提升站内用户活跃度。
- **目标**：活动期间参与测试 UV ≥ 500 万；测试完成率 ≥ 75%；结果页分享率 ≥ 15%；通过站内联动（赛后推送、竞猜联动）拉动赛事内页日活 UV 提升 5%。
- **范围**：H5 活动页（APP 内 WebView），不涉及客户端原生改动。站内联动（推送、竞猜彩蛋）由各业务模块配合实现。

---

## 二、页面/功能模块总览

本次活动包含 5 个核心页面/状态：

| 编号 | 页面/模块 | 简述 |
|------|----------|------|
| ① | 活动落地页 | 活动介绍 + 开始测试入口 |
| ② | 答题页 | 16 道题目逐题展示，含进度条 |
| ③ | 结果计算过渡页 | 答题完毕后的计算动效页，2~3 秒 |
| ④ | 结果页 | 人格展示 + 雷达图 + 分享入口 |
| ⑤ | 分享海报 | 一键生成的人格海报图片 |

**用户流程**：

```
活动入口（APP 多点位） → ① 落地页 → ② 答题页（16 题）→ ③ 过渡页 → ④ 结果页
 ↓
 ⑤ 分享海报
 ↓
 扫码跳转 APP
```

### 模块 ② — 答题页

| 功能模块 | 序号 | 需求内容 | 需求描述 |
|---------|------|---------|---------|
| 答题页 | 2.1 | 题目数量 | 共 16 道题目，按固定顺序逐题展示（题 1 → 题 16）。不支持随机打乱。 |
| | 2.2 | 题目结构 | 每题包含：题目序号（如 1/16）、题干文字、3 个选项（A/B/C）。每个选项包含选项标记和选项内容文字。 |
| | 2.3 | 选项分值 | 每题选项 A = 1 分，选项 B = 2 分，选项 C = 3 分。分值不展示给用户，仅用于后端计算。 |
| | 2.4 | 进度条 | 页面顶部展示答题进度条。格式：进度百分比 + 「N/16」文字。每答完一题，进度条以动画形式增长至下一位置（动画时长 300ms，easing: ease-out）。 |
| | 2.5 | 题目切换 | 用户选择选项后，当前选项高亮 0.3 秒，自动切入下一题。切换动效：当前题左滑出 + 新题右滑入，动画时长 400ms。 |
| | 2.6 | 回退操作 | 支持点击页面左上角返回按钮回到上一题。回退时保留之前的选项选择状态。用户可重新选择。 |
| | 2.7 | 最后一题 | 第 16 题选择后，自动跳转至结果计算过渡页（模块③）。 |
| | 2.8 | 退出中断 | 用户在答题过程中关闭/退出页面，已答题目数据缓存在本地（localStorage）。24 小时内重新进入，弹出 Toast：「检测到未完成的测试，继续上次进度？」，提供「继续」和「重新开始」两个选项。超过 24 小时，缓存清除，从头开始。 |

### 模块 ④ — 结果页

| 功能模块 | 序号 | 需求内容 | 需求描述 |
|---------|------|---------|---------|
| 结果页 | 4.1 | 页面结构 | 从上到下依次为：人格代号+中文昵称、**人物形象插画区域（含标签云）**、球迷人格解析（完整标签+深度解析文案）、球星对照/天敌/拍档、二维码、底部悬浮操作栏。 |
| | 4.2 | 人格代号 | 居中大字展示 4 字母英文代号（如 RIOT），下方为中文昵称（如 易燃易爆）。字号层级：代号 > 昵称。 |
| | 4.3 | 人格形象 | 居中展示人物形象插画（采用极客蓝扁平化风格）。插画容器下方展示该人格的 **“一句话标签组”**（如：战术极端分子 / 信仰守护者），标签采用色块加标签云布局。 |
| | 4.4 | 球迷人格解析 | 展示小标题「—— 球迷人格解析 ——」，下方完整呈现该人格的核心解析文案（即数据表中的“一句话标签”长段落）。文案采用左侧边框及斜体样式，增强结论感。 |
| | 4.5 | 底部二维码 | 在页面最下方展示活动二维码与 Slogan / 直播吧 logo。 |
| | 4.6 | 底部操作栏 | 页面最底部固定悬浮区域。第一行并排展示两个主操作按钮：**「保存相册」** 和 **「分享名片」**。按钮下方居中展示小字链接：**「重新预测」**。 |
| | 4.7 | 结果持久化 | 用户测试结果（人格代号 + 8 维评分）上报后端，与用户 UID 关联存储。同一用户重新测试后覆盖旧结果。 |

**交互规则**：

| 序号 | 交互行为 | 规则 |
|------|---------|------|
| 4.1 | 页面滚动 | 页面支持纵向滚动。底部分享按钮栏固定悬浮，不随页面滚动。 |
| 4.2 | 天敌/拍档点击 | 天敌人格、最佳拍档的名称不可点击（本期不做跳转查看其他人格详情）。 |

**异常处理**：

| 异常类型 | 展示规则 |
|---------|---------|
| 计算结果异常（无法匹配人格） | 兜底至默认人格 MOOD（墙头球迷），上报异常日志。 |
| 结果上报后端失败 | 本地缓存结果，结果页正常展示。后台静默重试 3 次（间隔 2s/4s/8s），仍失败则放弃上报，不影响用户体验。 |
| 结果上报后端失败 | 本地缓存结果，结果页正常展示。后台静默重试 3 次（间隔 2s/4s/8s），仍失败则放弃上报，不影响用户体验。 |

## 四、人格匹配算法（数据规则）

### 4.1 评分计算

```
步骤 1：收集 16 题用户选择，每题得分 1/2/3
步骤 2：按维度分组求和（每维度 2 题，合计 2~6 分）
 - F1 = 题1 + 题2
 - F2 = 题3 + 题4
 - E1 = 题5 + 题6
 - E2 = 题7 + 题8
 - S1 = 题9 + 题10
 - S2 = 题11 + 题12
 - A1 = 题13 + 题14
 - A2 = 题15 + 题16
步骤 3：映射为等级
 - 2~3 分 → L（Low）→ 数值 1
 - 4 分 → M（Mid）→ 数值 2
 - 5~6 分 → H（High）→ 数值 3
步骤 4：生成用户向量 [F1, F2, E1, E2, S1, S2, A1, A2]
```

### 4.2 隐藏人格优先检测

在执行向量匹配前，先检测是否触发隐藏人格：

| 隐藏人格 | 代号 | 触发条件 | 优先级 |
|---------|------|---------|-------|
| 杠精球评家 | TROL | 题 3 选 A（专业怼） + 题 12 选 C（开麦反击） + 题 9 选 A（社恐）= 线上键盘侠组合 | 「我不是故意抬杠，我只是觉得这个世界的观点太单一了。如果不跟我过两招，你怎么知道自己的热爱有多深？我毕生的追求，就是在对攻中寻求真理。」 |
| 四年一醒 | HERO | 题 13 选 A（佛系备战） + 题 16 选 C（凌晨3点必看） + 题 7 选 C（绝不放弃）= 矛盾体 | 「平时我是个路人甲，世界杯开始的那一刻，我能比任何死忠都更疯狂。这种四年一次的觉醒，懂的人自然懂。我不是在追星，我是在寻找那个丢失的自己。」 |

若触发隐藏人格，直接返回对应人格，跳过向量匹配。若同时满足 TROL 和 HERO 条件，优先返回 TROL。

### 4.3 向量匹配（曼哈顿距离）

```
预设人格原型向量（16 种）：
 GOAT = [1, 1, 2, 3, 2, 2, 3, 2]
 FANA = [1, 2, 3, 3, 2, 2, 3, 2]
 GURU = [2, 1, 1, 2, 1, 2, 2, 2]
 SOFA = [2, 2, 2, 2, 2, 3, 2, 1]
 RIOT = [2, 2, 3, 3, 2, 3, 2, 2]
 MUTE = [1, 2, 2, 2, 1, 1, 2, 2]
 YOLO = [2, 2, 2, 2, 2, 2, 2, 3]
 VIBE = [3, 3, 3, 2, 3, 3, 2, 2]
 SALT = [2, 2, 2, 2, 2, 3, 2, 2]
 JINX = [2, 2, 2, 1, 2, 2, 2, 3]
 MOOD = [3, 2, 2, 2, 2, 2, 1, 2]
 CLIP = [3, 3, 1, 1, 2, 1, 1, 1]
 CHEF = [3, 3, 2, 2, 3, 2, 2, 1]
 MONK = [3, 3, 1, 1, 1, 1, 1, 1]
 SOUL = [2, 2, 3, 1, 2, 2, 3, 2]
 DRIP = [2, 3, 3, 2, 2, 2, 2, 2]

匹配公式：
 距离 = Σ|用户向量[i] - 人格原型向量[i]| (i = 1~8)
 取距离最小的人格为结果
 若多个人格距离相同，按人格表顺序优先（GOAT > FANA > ... > DRIP）
```

### 4.4 全部计算在前端完成

- 计算逻辑封装在前端 JavaScript 中，不依赖后端接口。
- 结果产出后，将用户 UID + 人格代号 + 8 维原始分 + 时间戳上报后端存储（异步，不阻塞页面展示）。
