# Technical Notes - 世界杯球迷人格大鉴赏（WFTI）H5 活动

## 1. 技术方向
- 前端：H5 / mobile-web，运行于直播吧 App WebView
- 设计产物：HTML 高保真 mockups
- 计算逻辑：前端本地完成，不依赖实时计算接口
- 后端职责：结果存储、历史查询、联动消费数据提供

## 2. 前端实现重点
### 2.1 状态管理
- 管理题目索引、已选答案、八维得分、结果数据、登录态与活动态
- 本地缓存结构需包含：答题进度、选项答案、最近保存时间
- 缓存有效期 24 小时，超时自动失效

### 2.2 人格计算引擎
- 16 题答案映射为 1/2/3 分
- 先执行隐藏人格检测，再执行 16 种常规人格匹配
- 结果对象需至少包含：人格代号、昵称、8 维原始分、维度等级、时间戳
- 算法模块需可独立单测

### 2.3 海报生成
- 使用 Canvas/html2canvas 生成 PNG
- 生成期间显示 loading 蒙层
- 超过 5 秒未完成时提示失败并允许重试
- 结果页截图前需隐藏底部操作栏

### 2.4 视觉落地关键点
- 设计与开发需使用同一套 KV 底部颜色变量，避免 1px seam
- 卡片层叠高度、底部环境渐变、粒子装饰与 Logo 锚点需按设计规范实现
- 页面需支持长屏设备下的底部压舱式布局，不得出现无设计大面积空白

## 3. 宿主与接口依赖
### 3.1 宿主桥接待确认
- 登录唤起能力
- 系统分享面板调起能力
- 保存图片到相册权限与回调能力
- APP 内活动页路由与二维码落地页跳转能力

### 3.2 服务接口待确认
- 查询用户历史测试结果
- 上报/覆盖用户测试结果
- 生成或下发结果页二维码目标链接
- 站内联动消费人格结果的接口或数据同步机制

## 4. 数据结构建议
### 4.1 结果上报字段
- uid
- personality_code
- personality_name
- answers
- raw_scores
- level_vector
- created_at
- client_env

### 4.2 历史结果查询字段
- uid
- latest_personality_code
- latest_personality_name
- raw_scores
- updated_at

## 5. 测试关注点
- 16 题切题动画与回退逻辑正确性
- 本地缓存恢复与过期清理
- 隐藏人格优先级正确性
- 常规人格距离计算与并列优先顺序
- 结果上报失败重试不影响展示
- 海报生成兼容性与权限提示
- 活动未开始、已结束、网络异常等边界态
