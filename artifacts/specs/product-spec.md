# Product Spec - 世界杯球迷人格大鉴赏（WFTI）H5 活动

## 1. 产品概述
一个嵌入直播吧 App WebView 的世界杯主题 H5 活动。用户通过 16 道测试题获得世界杯球迷人格结果，并查看可分享的结果页与海报内容。活动页不依赖新增原生开发，前端完成测试主链路与人格计算，后端负责结果接收与存储。

## 2. 业务目标
- 承接世界杯赛期流量峰值，提升站内活动参与度
- 以人格结果与海报分享增强传播与社交表达
- 通过活动结果与站内联动提升赛事详情页访问与留存

## 3. 页面范围
### 3.1 活动落地页
- 页面标题：WFTI — 世界杯球迷人格大鉴赏
- Slogan：你不是在看世界杯，你是在演你自己。
- 活动说明：16 道题，3 分钟，测出你的世界杯球迷人格密码
- 固定底部 CTA：开始鉴赏
- 已测试用户展示：查看我的结果 + 重新测试
- 未登录点击开始测试时，先拉起登录弹窗

### 3.2 答题页
- 共 16 道题，固定顺序
- 每题展示题号、题干、3 个选项
- 顶部进度条显示百分比与 `N/16`
- 选项选中后自动切题
- 支持回退到上一题并保留已选状态
- 中断退出后，24 小时内支持继续作答

### 3.3 结果计算过渡页
- 固定展示 2.5 秒
- 展示计算动效与「正在分析你的球迷DNA……」文案
- 前端在此阶段完成 8 维评分与人格匹配

### 3.4 结果页
- 人格代号 + 中文昵称
- 人物形象插画区域 + 标签云
- 球迷人格解析区
- 球星对照 / 天敌 / 拍档信息区
- 底部二维码与品牌信息
- 固定底部操作栏：保存相册、分享名片、重新预测

### 3.5 分享海报
- 由结果页生成 PNG 长图
- 包含人格代号、昵称、一句话标签、八维雷达图、画像文字、球星对照、二维码、logo
- 支持系统分享与保存图片

## 4. 题目与人格规则
### 4.1 答题规则
- 共 16 题，A/B/C 分别记 1/2/3 分
- 按 F1/F2/E1/E2/S1/S2/A1/A2 八个维度分组
- 每个维度合计得分 2~6 分，再映射为 L/M/H 即 1/2/3
- 输出用户向量 `[F1, F2, E1, E2, S1, S2, A1, A2]`

### 4.2 隐藏人格
- TROL：题3=A + 题12=C + 题9=A，优先级最高
- HERO：题13=A + 题16=C + 题7=C
- 若同时满足，优先返回 TROL

### 4.3 常规人格匹配
- 使用预设 16 种人格原型向量
- 采用曼哈顿距离求最近人格
- 若距离相同，按人格表顺序优先返回
- 匹配失败时兜底为 MOOD，并记录异常日志

## 5. 数据持久化与联动
### 5.1 结果上报
- 前端异步上报：UID + 人格代号 + 8 维原始分 + 时间戳
- 同一用户重新测试后覆盖旧结果
- 失败时本地缓存，并按 2s/4s/8s 静默重试 3 次

### 5.2 站内联动定义
- 世界杯专题页入口卡片
- 赛后个性化推送
- 竞猜区 JINX 彩蛋浮层
- 社区热帖底部引导浮层
- 个人主页人格标签显示/隐藏

## 6. 视觉与设计约束
- 设计必须以用户提供的 `artifacts/source-design/design-spec.md` 为硬约束
- 已确认 KV 风格为 **A 高阶 3D 风格建模**
- 视觉要求包括但不限于：
  - 全局高亮影棚调性
  - KV 宽度锁定 390px，高度弹性
  - 标题色彩与 KV 底色融合，禁止绝对黑
  - 所有页面背景与 KV 底部像素色对齐
  - 卡片大圆角 + 负边距层叠
  - 页面底部禁止大面积真空留白
  - CTA 使用拟真微渐变与弥散阴影体系
  - Logo 按规范从素材库自适应选型

## 7. 异常与边界
- 页面加载失败时展示通用错误兜底页
- 活动未开始/已结束时展示状态页
- 网络异常时保留本地进度并提示重试
- 本地缓存读取失败时静默回到第 1 题
- Canvas 渲染连续失败 3 次后，隐藏生成海报按钮，仅保留保存截图提示

## 8. 非功能要求
- H5 需适配主流移动端长屏 WebView 环境
- 首屏视觉覆盖率需满足长屏下 ≥ 85%
- 页面底部环境色与 KV 底色需保持像素级连续性
- 交互动画需保证流畅，避免卡顿与闪屏
- 辅助文案颜色不得浅于 #94A3B8
