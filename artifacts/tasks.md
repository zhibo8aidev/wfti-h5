# WFTI H5 任务拆解

## 前端

### FE-01
- 编号：`FE-01`
- 目标：搭建 H5 工程骨架、页面状态机与设计 token，确保运行于直播吧 App WebView 的 mobile-web 基础环境。
- 关键产出：项目目录骨架、页面路由状态枚举、全局样式变量、活动态/异常态外壳页、骨架屏方案。
- 依赖：`artifacts/technical-design.md`、`artifacts/design.md`、`artifacts/react_native_mockups/index.html`
- 风险或待确认项：WebView 环境初始化信息获取方式 `【待桥接确认】`

### FE-02
- 编号：`FE-02`
- 目标：落地题库与人格展示静态资源模块，建立可配置、可复用的数据基线。
- 关键产出：16 题题库配置、8 维映射表、16 种常规人格原型、隐藏人格规则、人格展示元数据文件。
- 依赖：已批准 PRD、设计稿文案、前端工程骨架
- 风险或待确认项：部分人格展示文案/球星对照素材是否已有最终结构化源数据，需产品/内容确认

### FE-03
- 编号：`FE-03`
- 目标：实现前端本地人格计算引擎，支持隐藏人格优先级、常规人格匹配与异常兜底。
- 关键产出：`personality-engine` 模块、输入输出类型、单元测试用例、异常日志埋点钩子。
- 依赖：`FE-02`
- 风险或待确认项：16 种人格原型表最终内容需与产品确认，不得开发自行补完

### FE-04
- 编号：`FE-04`
- 目标：实现活动落地页与登录前置流程，支持未测/已测两种入口态。
- 关键产出：落地页 UI、已测试用户入口分支、登录拦截逻辑、活动未开始/已结束状态页。
- 依赖：`FE-01`、`FE-06`、`BE-02`、`BR-01`
- 风险或待确认项：登录拉起协议与登录成功回调 `【待桥接确认】`；历史结果查询接口 `【待接口确认】`

### FE-05
- 编号：`FE-05`
- 目标：实现 16 题答题主链路，包含固定顺序作答、自动切题、回退、断点续答。
- 关键产出：答题页 UI、进度条、切题动画、返回上一题、24 小时本地草稿恢复弹层。
- 依赖：`FE-01`、`FE-02`
- 风险或待确认项：本地存储上限与 WebView 对 `localStorage` 的兼容边界需实机验证

### FE-06
- 编号：`FE-06`
- 目标：实现结果计算过渡页、结果页渲染与异步结果上报闭环。
- 关键产出：2.5 秒过渡页、结果页首屏与续读区、结果写入重试机制、本地待同步队列。
- 依赖：`FE-03`、`FE-05`、`BE-01`
- 风险或待确认项：结果写入接口字段与覆盖语义 `【待接口确认】`

### FE-07
- 编号：`FE-07`
- 目标：实现海报生成能力，满足保存相册、分享名片前的图片产出需求。
- 关键产出：海报专用模板、资源预加载、Canvas/html2canvas 生成、超时与失败回退处理。
- 依赖：`FE-06`、`BR-02`、`BE-03`
- 风险或待确认项：目标 WebView 对 `html2canvas` 的兼容性、图片保存能力 `【待桥接确认】`

### FE-08
- 编号：`FE-08`
- 目标：实现二维码区与站内跳转入口，收口外部依赖为 adapter。
- 关键产出：二维码展示组件、静态/动态链接适配、按钮跳转封装、失败兜底提示。
- 依赖：`FE-06`、`BR-04`、`BE-03`
- 风险或待确认项：二维码链接来源、站内路由协议 `【待接口确认】` `【待桥接确认】`

### FE-09
- 编号：`FE-09`
- 目标：实现前端 mock / adapter 层，保证未知接口与桥接不阻塞 H5 主链路开发。
- 关键产出：`resultRepository`、`hostCapabilityService`、web mock 数据、环境切换方案。
- 依赖：`FE-01`
- 风险或待确认项：真实接口返回结构不明确，adapter 字段可能需二次对齐 `【待接口确认】`

## 后端

### BE-01
- 编号：`BE-01`
- 目标：提供最小可用的结果存储接口，支持同一用户结果覆盖更新。
- 关键产出：`POST /api/wfti/result`、参数校验、幂等/覆盖逻辑、错误码定义。
- 依赖：`artifacts/technical-design.md`
- 风险或待确认项：认证方式、网关路径、鉴权头格式 `【待接口确认】`

### BE-02
- 编号：`BE-02`
- 目标：提供历史结果查询接口，支撑落地页“查看我的结果”和登录后直达结果页。
- 关键产出：`GET /api/wfti/result/latest`、无结果/有结果返回结构、接口文档。
- 依赖：`BE-01`
- 风险或待确认项：是否由网关透传 `uid` 或由后端根据登录态解析用户身份 `【待接口确认】`

### BE-03
- 编号：`BE-03`
- 目标：提供二维码链接与分享配置的最小配置能力，避免前端硬编码不可控。
- 关键产出：二维码落地链接配置接口或静态配置文件、分享标题/文案/链接模板。
- 依赖：`BE-01`
- 风险或待确认项：二维码是否需要动态短链、是否由运营后台维护 `【待接口确认】`

### BE-04
- 编号：`BE-04`
- 目标：定义站内联动消费边界，确保世界杯专题、推送、竞猜、社区等业务可取到人格结果。
- 关键产出：联动消费字段说明、查询/同步方案建议、数据口径说明。
- 依赖：`BE-01`、`BE-02`
- 风险或待确认项：消费方式是接口拉取还是异步同步尚未定稿 `【待接口确认】`

### BE-05
- 编号：`BE-05`
- 目标：提供前后端并行开发所需的 mock 服务或测试环境。
- 关键产出：mock 数据、测试环境部署说明、联调账号/样例结果。
- 依赖：`BE-01`、`BE-02`、`BE-03`
- 风险或待确认项：测试环境可用性与发布节奏需后端确认

## 桥接 / 配置 / 运营支持

### BR-01
- 编号：`BR-01`
- 目标：确认宿主登录 bridge 能力，并沉淀 H5 适配协议。
- 关键产出：登录调用方式、成功/取消回调结构、H5 adapter 协议文档。
- 依赖：宿主客户端/中间层协同
- 风险或待确认项：登录拉起能力是否现成可用 `【待桥接确认】`

### BR-02
- 编号：`BR-02`
- 目标：确认分享与保存相册 bridge 能力，覆盖海报分享链路。
- 关键产出：分享面板调起协议、图片分享/链接分享能力说明、保存相册权限与回调文档。
- 依赖：宿主客户端/中间层协同、`FE-07`
- 风险或待确认项：是否支持直接分享图片、保存图片是否需额外白名单 `【待桥接确认】`

### BR-03
- 编号：`BR-03`
- 目标：确认活动状态、分享文案、二维码链接等配置的提供方式。
- 关键产出：配置来源说明、字段清单、更新机制、默认兜底值。
- 依赖：后端/运营协同、`BE-03`
- 风险或待确认项：配置是否走现有活动配置中心 `【待接口确认】`

### BR-04
- 编号：`BR-04`
- 目标：确认站内路由跳转桥接，用于二维码与结果页内联动入口。
- 关键产出：路由 schema/path、参数格式、失败兜底链接规则。
- 依赖：宿主客户端/中间层协同、`BE-03`
- 风险或待确认项：App 内跳转协议是否统一、扫码后落地行为是否可控 `【待桥接确认】`

### BR-05
- 编号：`BR-05`
- 目标：提供运营所需的最小支持项，确保入口卡片、分享文案、二维码素材可上线。
- 关键产出：活动标题/摘要/分享默认文案、二维码目标页说明、素材提审清单。
- 依赖：产品、运营、设计、`BE-03`
- 风险或待确认项：运营素材最终交付时间可能影响海报和分享内容定版

## 埋点

### TR-01
- 编号：`TR-01`
- 目标：定义 H5 全链路埋点方案，覆盖落地页、答题、结果、海报、桥接交互。
- 关键产出：埋点事件清单、字段字典、触发时机说明。
- 依赖：`artifacts/technical-design.md`
- 风险或待确认项：现有站内埋点 SDK 在 WebView 中的初始化方式 `【待桥接确认】`

### TR-02
- 编号：`TR-02`
- 目标：完成前端埋点接入，并确保失败不影响主链路。
- 关键产出：埋点封装层、页面与行为事件上报、异常日志捕获。
- 依赖：`TR-01`、`FE-04`、`FE-05`、`FE-06`、`FE-07`
- 风险或待确认项：埋点 SDK 是否支持离线补发需确认

### TR-03
- 编号：`TR-03`
- 目标：打通后端结果写入与联动消费侧的日志/监控，便于核对覆盖更新与数据一致性。
- 关键产出：接口访问日志、结果覆盖日志、关键错误告警项。
- 依赖：`BE-01`、`BE-02`、`BE-04`
- 风险或待确认项：日志平台和告警归属需后端确认

## 联调与验收

### QA-01
- 编号：`QA-01`
- 目标：完成前后端联调，验证“查询历史结果 -> 作答 -> 本地计算 -> 结果上报覆盖 -> 回查展示”闭环。
- 关键产出：联调记录、接口字段核对单、覆盖更新验证结果。
- 依赖：`FE-06`、`BE-01`、`BE-02`、`FE-09`
- 风险或待确认项：接口鉴权和用户身份透传未确认 `【待接口确认】`

### QA-02
- 编号：`QA-02`
- 目标：完成 bridge 联调，验证登录、分享、保存相册、路由跳转四类宿主能力。
- 关键产出：桥接联调记录、异常回调验证、降级策略验证。
- 依赖：`BR-01`、`BR-02`、`BR-04`、`FE-07`、`FE-08`
- 风险或待确认项：宿主环境测试包与联调窗口期 `【待桥接确认】`

### QA-03
- 编号：`QA-03`
- 目标：完成设计验收与多机型 WebView 回归，确保长屏布局、防真空、底栏安全区与海报表现符合已批设计。
- 关键产出：设计走查清单、390x844/430x932 验收截图、海报生成兼容性报告。
- 依赖：`FE-04`、`FE-05`、`FE-06`、`FE-07`
- 风险或待确认项：低端机性能与字体渲染差异可能影响最终表现

### QA-04
- 编号：`QA-04`
- 目标：完成上线前验收，覆盖活动状态页、网络异常、断点续答、结果兜底、海报失败、接口失败重试等边界。
- 关键产出：验收 checklist、边界测试记录、上线阻塞项清单。
- 依赖：全部核心开发任务
- 风险或待确认项：活动开始/结束时间配置、线上域名与白名单信息 `【待接口确认】`

## 推荐排期顺序
1. 前端优先启动 `FE-01`、`FE-02`、`FE-03`、`FE-09`。
2. 后端并行启动 `BE-01`、`BE-02`、`BE-03`，先提供 mock/测试环境。
3. 页面主链路推进 `FE-04`、`FE-05`、`FE-06`。
4. bridge 与路由确认并行推进 `BR-01`、`BR-02`、`BR-04`。
5. 主链路稳定后完成 `FE-07`、`FE-08`、`TR-02`。
6. 最后执行 `QA-01` 至 `QA-04` 收口。
